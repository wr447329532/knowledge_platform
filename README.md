文件共享和知识管理平台
=================================

本项目是在 `vscode` 目录下从 0 开发的**文件共享和知识管理平台**，采用 **Python + FastAPI** 后端与 **Vue 3 + Vite** 前端。  
**最后更新：2026 年 3 月 15 日（含版本匹配弹窗整合、关键词提取增强）**

**当前已实现：**

- 用户系统：**邮箱登录**、管理员创建用户（邮箱必填，用户名仅用于显示）、强密码策略（8 位以上，含大小写、数字、特殊字符）
- 文件库：创建 / 编辑 / 删除 / 软删除（回收站），按文件库管理文件；支持**部门库**（所属部门成员可访问）
- 文件上传 / 下载 / 版本历史 / 重命名 / 回收站（恢复、彻底删除，超过 30 天自动清理）
- **版本匹配上传**：上传前自动提取关键词并搜索库内可能相关文件；有结果时在上传弹窗内进行版本确认（作为新版本或新文件），无结果直接上传；支持多文件队列逐一确认
- 在线预览：点击文件名在新窗口预览，支持图片、PDF、文本（txt/md/json 等）
- **部门管理**：部门树、部门库、新建子部门、负责人展示、部门搜索
- **共享文件库**：基于库级 ACL 的「我分享的文件库 / 分享给我的文件库」视图，统一展示库名称、共享范围（公开/部门/指定成员）、我的权限（读写/只读）
- **共享文件（单文件）**：在文件列表中对单个文件发起分享，支持只读/可下载权限（用于细粒度协作）
- **我的文件库**：列表与网格视图、排序（最近修改/文件名/大小/创建时间）、三点操作菜单、长文件名省略
- **统一权限模型（库级 ACL）**：
  - 访问优先级：超级管理员 › 库拥有者 › 库成员 › 部门库成员 › 公开库
  - `Library.visibility = private / members / department / public` 控制库可见范围
  - `LibraryMember` 维护库成员及角色（只读 / 读写）；`allow_download` 控制非拥有者是否可下载
  - 后端 `has_library_access / can_access_file / can_download_file / get_accessible_library_ids` 统一判定访问与下载权限
  - 修复了「无部门用户误看到全部部门库」的权限漏洞（无部门或无效部门的普通用户不再拥有全局访问）
- **账户管理（普通用户）**：
  - 侧边栏底部新增「账户」入口，普通用户可进入账户设置页
  - 个人信息：显示并支持修改姓名，邮箱和所属部门只读（由管理员维护），保存后即时生效并有成功提示
  - 安全设置：支持修改密码（校验原密码 + 强密码规则），以及查看基础安全信息（账户状态、上次登录/修改密码时间占位）
- 审计日志：记录上传、删除、恢复、分享、预览、下载等操作，支持按用户与操作类型模糊筛选（仅管理员可查）
- 系统管理：
  - **用户管理**：启用/停用、重置密码、分配部门、管理员标记
  - **部门管理**：树形表格、存储统计占位、部门负责人
  - **存储管理（新）**：按部门 / 用户 / 文件类型聚合存储使用，支持配额调整
  - **通知管理（新）**：通知模板、发送历史、通知开关、管理员一键群发通知

**后续规划：** 新建资料库更细粒度权限设置（角色 / 继承策略）、全文搜索、本地 LLM 知识问答等。

快速开始
--------

> ⚠️ **强制安全要求：必须修改 JWT_SECRET_KEY**
>
> 后端使用 JWT 进行身份认证，密钥由 `.env` 文件中的 `JWT_SECRET_KEY` 控制。  
> **如果保留默认值 `change_this_secret_in_env`，后端会在启动时直接报错退出，拒绝运行。**  
> 请在部署前务必：
> - 在项目根目录创建或修改 `.env` 文件；
> - 设置一个足够随机、长度较长的 `JWT_SECRET_KEY`（例如 32 位以上随机字符串）；
> - 重启后端服务。

**注意：需先启动后端，再启动前端，最后在浏览器打开前端地址。**

**1. 启动后端**

在项目根目录 `knowledge_platform` 下，使用已安装依赖的 Python 环境（如 conda）：

```bash
cd knowledge_platform
conda activate finance_agent   # 或你的环境，需已 pip install -r requirements.txt
uvicorn backend.app.main:app --reload --host 0.0.0.0
```

或使用脚本（会自动用 finance_agent 环境）：

```bash
cd knowledge_platform
./run_backend.sh
```

后端默认运行在：`http://127.0.0.1:8000`  
接口文档：`http://127.0.0.1:8000/docs`

**2. 启动前端**

新开一个终端：

```bash
cd knowledge_platform/frontend
npm install    # 首次需要
npm run dev
```

前端默认运行在：`http://localhost:5173`

**3. 登录**

浏览器打开 **http://localhost:5173**，使用默认管理员账号：

- 邮箱：`admin@example.com`（或数据库中为该账号配置的邮箱）
- 密码：`admin123`（首次使用建议通过系统管理修改为强密码）

**4. 常用地址**

- 前端页面：http://localhost:5173
- 后端健康检查：http://127.0.0.1:8000/health
- 接口文档（Swagger）：http://127.0.0.1:8000/docs
